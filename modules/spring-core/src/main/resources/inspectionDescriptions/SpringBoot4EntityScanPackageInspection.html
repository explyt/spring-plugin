Reports the legacy @EntityScan import location in Spring Boot 4+ projects and offers a quick-fix that rewrites the import to the new package.

In Spring Boot 4.0 @EntityScan moved from org.springframework.boot.autoconfigure.domain to org.springframework.boot.persistence.autoconfigure. The inspection only triggers when Spring Boot 4+ is detected and the new class is resolvable on the classpath.

Example:


// before
import org.springframework.boot.autoconfigure.domain.EntityScan;

// after
import org.springframework.boot.persistence.autoconfigure.EntityScan;

See the Spring Boot 4.0 Migration Guide.
