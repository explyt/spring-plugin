Reports referenced bean is incorrect in the value parameter of the @Lookupannotation.

Example:


  package com;

  @Configuration
  LookupConfiguration {
    @Bean
    public BeanA beanA {...}
    @Bean
    public BeanB beanB {...}
  }

  @Component
  class MyComponent {
    @Lookup("beanA")
    public BeanA myBeanA() {...}

    @Lookup("beanA") // reports "Expected bean of 'com.BeanB' type but found 'com.BeanA'"
    public BeanB myBeanB() {...}

    @Lookup("beanNotFound") // reports "Can not resolve bean 'beanNotFound'"
    public OtherBean otherBean() {...}
  }
