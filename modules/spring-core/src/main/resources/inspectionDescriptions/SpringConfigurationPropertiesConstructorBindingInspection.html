Reports a redundant @ConstructorBinding annotation on a single-constructor @ConfigurationProperties class.

Since Spring Boot 3.0 @ConstructorBinding is no longer needed when a class (or record) has a single constructor: such a class is bound through its constructor automatically. The annotation should only be used when there are multiple constructors and one of them must be selected for property binding.

Example:


@ConfigurationProperties(prefix = "some.prefix")
data class SomeProperties @ConstructorBinding constructor( // @ConstructorBinding is redundant
        val first: String,
        val second: Long
)

The quick-fix removes the redundant annotation.
